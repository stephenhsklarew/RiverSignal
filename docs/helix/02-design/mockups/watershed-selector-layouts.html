Watershed selector — scalable layouts

Four options to replace the horizontal .watershed-tabs strip, which runs out of room past ~6 watersheds. All stay anchored top-right over the map and follow a state → watershed drill-down. Everything below is interactive — click around.

Today — flat horizontal strip

One button per watershed in a single row. Already crowded at 9; breaks the corner layout as more states come online.

Option A — Cascading dropdowns

Smallest footprint. Two compact dropdowns: pick a State, then the Watershed list filters to that state. Familiar pattern, almost no corner real-estate, scales to any number of states. Watershed dropdown stays disabled until a state is chosen.

Best when: corner space is tight and states will keep growing. Trade-off: two taps, and you can't see all rivers at a glance.

Option B — Master / detail popover

Recommended. A single pill shows the current selection (Oregon · McKenzie). It opens a two-column panel: states on the left with a count badge, watersheds for the hovered/selected state on the right. One open surface, fast scanning, scales cleanly.

Best when: you want one elegant control that reads the current context and scales to dozens of rivers. Trade-off: slightly wider panel than A.

Option B — before any state is selected (first load)

On first load — or after "View all rivers" — nothing is selected and the map shows every watershed. The pill reads "All rivers" with a muted marker. Open it and the left column lists states (none active); the right column shows a hint until you pick/hover a state. A persistent "Show all rivers" row at the bottom lets you return to the unfiltered view any time.

Touch note: hover previews a state's rivers on desktop; on mobile the first tap locks the state + fills the right column, the second tap picks the river.

Option C — Search-first modal

Scales furthest. A trigger opens a centered modal with a search box plus rivers grouped under state headings. Type "ska" to jump straight to Skagit, or browse by state. Color dots match the map markers.

Best when: the list grows past ~20 rivers and search matters most. Trade-off: a modal is heavier than an inline control for small lists.

Option D — State tabs + watershed chips

Most visible. A small inline card: state pills on top, watershed chips for the active state below. Everything for one state is visible at once — no second click to see options. Good middle ground while state count stays modest.

Best when: a handful of states, and seeing every river for a state matters. Trade-off: the card grows tall/wide; state row scrolls horizontally past ~8 states.
